## Learned User Preferences

- Keep the interface free of shadows. Motion should feel slow, smooth, and heavy—long easing curves and opacity/filter interpolation only. No bounce, springy overshoot, scale-in, or slide-in entrances.
- Treat the site background as solid white only; avoid extra fill colors on panels or cells unless placeholder media explicitly keeps its fill.
- The top band or header should stay sticky to the top edge, stay visually transparent (no background fill), and sit above scrolling content. On home and `/info` below `lg`, `SiteBrandStrip` fixes “Daniel Derro” and “No-School Studio Records” at the top (same pattern as info); use `hideBrandBelowLg` on `SiteFooter` so those labels are not duplicated in the footer band.
- Pin the site footer to the bottom of the viewport and give the main page enough bottom padding so content can scroll above it (including safe-area where relevant); keep the footer visually transparent with no opaque background fill (consistent with the header band). Footer navigation should read as equal-width items within their grid span, with labels left-aligned inside each cell; hover should invert to black background with white text and no visible border (use `border-transparent` on hover). Footer brand labels ("Daniel Derro", "No-School Studio Records") should always link to home (`/`). Below `lg` (1024px), right-align "No-School Studio Records" in the header band and footer (Daniel Derro stays left).
- For local checks, run the dev server on port 3000 via `npm run dev` (routes through `scripts/dev.sh` which kills stale listeners before launching Turbopack). Use `npm run dev:clean` to wipe `.next` first when manifest/cache errors appear.
- UI stack is Next.js with Tailwind, Framer Motion for animation, and Lenis for smooth scrolling; Sanity CMS at `/studio` is used for client-managed portfolio content. Motion tokens live in `src/lib/motion.ts`; the `.hover-smooth` utility class in `globals.css` handles CSS-driven hover transitions. Exhibitions and Radio nav placeholders show a hover crossfade to "Coming Soon" (primary red fill, white text) and render as non-navigating controls until those routes ship.
- On small screens, use uniform padding around the layout via `--spacing-margin` (12px at all breakpoints).
- The `InfoColumns` component powers both the sticky top panel and the bottom panel. Always keep all three inner text-grid slots (`repeat(3, 1fr)`) even when a column's content is hidden — render an empty `<div />` placeholder to hold the grid position. The six-column `page-grid` structure must never collapse. On the home page, keep Contact in the bottom row (fourth cell), not the top band; place About and Clients to the right of Contact; keep the bottom-row image cell empty. On mobile home only, About can render above the hero/bio images (`homeMobileAboutAboveHero` on `InfoColumns`); Clients stay below the images.
- On the home page, present the Sanity-fed gallery as a continuous six-column image grid without per-project title rows; show at most 30 images with each row filled; align rows to the top with uniform vertical gap between rows; clicking a gallery image navigates to that image’s project. When hovering a gallery image, non-hovered images use about 20% opacity and grayscale. The Daniel portrait in the top panel is shown in grayscale at 10% opacity.
- On the work page, hovering a project on tablet/desktop pops up a cursor-following image overlay. Each project preview is rendered by the `HoverPreview` subcomponent inside a fixed-inset wrapper; every filtered project's image stays mounted so only opacity animates between hover states, making rapid project-to-project hovering crossfade smoothly with no reload flicker or blank gap. Each preview is a fixed 200px wide and preserves the project image's true aspect ratio — intrinsic dimensions are parsed from the Sanity asset `_ref` (e.g. `image-<hash>-<w>x<h>-<ext>`) with a 4:3 fallback, and the image uses `object-fit: contain` so it is never cropped. Positioning is edge-aware via `useTransform` on shared pointer motion values: the preview prefers to sit above and to the right of the cursor with a 16px gap on both axes, but flips below (when near the top of the viewport) or to the left (when near the right edge) so top-row / right-edge projects are never clipped. A small `HOVER_PREVIEW_EDGE_SAFE` (8px) margin is included when deciding to flip. The opacity crossfade is intentionally snappy (~0.18s with `MOTION.ease.out`). The overlay hides (all previews fade to 0 together) when the pointer leaves the scrollable project list. The page background stays solid black — do not fill it with the hovered project's image. On mobile the overlay is omitted so the surface stays solid black. Project titles and clients use `text-caption` on the desktop/tablet grid and on mobile; tags use `text-meta`. On mobile, each project row uses two columns: left column has title and client on one line with 4px gap, tags on the line below with no extra gap, and the right column has a 40px-wide preview image (aspect ratio preserved), right-aligned. Lay out project blocks in a checkerboard on large viewports with empty cells between projects (row one: empty at columns 2, 4, 6; row two: empty at 1, 3, 5; repeat).
- Target responsive column counts: two columns below 768px, four columns from 768px to 1023px, six columns from 1024px up (aligned with `.page-grid` breakpoints).

## Learned Workspace Facts

- The app uses Next.js 15 (App Router), React 19, and Tailwind CSS v4 with `@tailwindcss/postcss`.
- Layout uses responsive `.page-grid` in `globals.css`: two columns by default, four from `min-width: 768px`, six from `min-width: 1024px`; `--spacing-margin` is 12px at all breakpoints; `--spacing-gutter` scales by breakpoint. The home gallery uses row spacing tied to `--spacing-gutter` so vertical gaps match horizontal column gutters (avoid a global `.page-grid` `row-gap: 0` rule that overrides intended row gaps). On the work page at `lg` and up, `.page-grid.work-project-grid-row` sets `column-gap: 56px` so project columns use the same horizontal rhythm as tablet (default `lg` gutter would otherwise be 120px).
- Earlier microsite code is archived under `src/archive/` rather than mixed into the current site surface. `tsconfig.json` excludes `src/archive` so archived code is not type-checked; `npm run build` can still fail if archived files import packages that are no longer dependencies.
- Global styles strip box shadows site-wide. Framer Motion handles animation (page transitions via `src/app/template.tsx`, hover/scroll via motion components). Lenis smooth scroll is initialized in `src/components/SmoothScroll.tsx` (lerp 0.07 for heavy feel). Both providers skip `/studio` routes.
- Stale or mixed `.next` output (for example after `next build` then `next dev`) can cause local Internal Server Errors or missing chunk errors; `npm run dev:clean` handles this automatically.
- Never leave large directories (e.g. `node_modules_old`) in the project root — they cause webpack/Turbopack file-watcher exhaustion and hang page compilation indefinitely. `.gitignore` now blocks `node_modules_old`.
- Embedded Sanity CMS powers portfolio projects and media; Studio is at `/studio`. Use versions aligned with Next.js 15 (for example `sanity@4` with `next-sanity@11`). Until video playback exists, gallery `videoAsset` entries should use thumbnails only as on-page stills. Public contact email shown in the site UI is `info@ns-sr.org`. Server routes use `sanityFetchOrDefault` in `src/sanity/lib/fetch-safe.ts` so failed Sanity fetches fall back to empty data or `notFound` instead of throwing 500s.
- The site has three page routes beyond home: `/info` (white background; non-section titles in red like the home page—not a full red inverted page), `/work` (dark 100vh project matrix with hover-swapped viewport-fixed background images on tablet/desktop and checkerboard empty cells on large viewports), and `/work/[slug]` (project detail with fixed side info and scrollable media column; the footer title/client block is a wrapping flex with `rowGap: 0` when lines stack). Work routes use `src/app/work/layout.tsx` with `data-work-surface`; `globals.css` sets `html`/`body` black when that marker is present so rubber-band overscroll does not flash the default white canvas.
- `SiteFooter` accepts `activePath` (highlights the matching nav item) and `inverted` (swaps to white foreground for dark backgrounds) props. Work and project detail share `SiteHeaderBand` for the top band. Client breakpoints use `useMediaQuery` in `src/lib/use-media-query.ts` (e.g. work grid chunking). Project detail slideshow cursor arrows and desktop split footer layout use `MEDIA_DESKTOP_FINE_POINTER` in `src/lib/media-queries.ts` (`(min-width: 1024px) and (pointer: fine)`) so wide viewports with touch (e.g. iPhone Safari “Request Desktop Website”) keep touch-first layout.
- `text-body` CSS token is 14px Helvetica Neue Medium with 1.25 line-height and 1% letter-spacing, defined in `globals.css`.
- `StickyHeroFadeImage` is a client component that uses a Framer Motion `MotionValue` to fade the home-page sticky hero image to transparent as the user scrolls near the document bottom.
